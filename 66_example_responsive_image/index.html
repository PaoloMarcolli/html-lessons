24 . Immagini Resposive

Fonte Immagine

 Le immagini responsive sono un elemento essenziale per garantire una buona esperienza utente su dispositivi con dimensioni dello schermo e risoluzioni diverse. Questo capitolo del Corso ti guiderà attraverso i vari metodi per rendere le immagini del tuo sito web responsive.

Parlando di design responsive, abbiamo imparato a utilizzare le media query per creare layout separati per dispositivi mobili, tablet e desktop. Ora vogliamo fare lo stesso con le immagini. Proprio come le media query ci permettono di presentare condizionatamente diverse regole CSS, vogliamo visualizzare diverse immagini in base al dispositivo dell'utente.

Le immagini hanno dimensioni intrinseche.Non possiamo allungare una foto di 500×250 pixel oltre i 500 pixel di larghezza senza che diventi pixelata. I display retina e i dispositivi mobili complicano ulteriormente le cose. Per rendere le nostre immagini responsive, dobbiamo considerare tre aspetti:

  • Le dimensioni del dispositivo
  • Le dimensioni dell'immagine
  • La risoluzione dello schermo del dispositivo

Per sperimentare con le immagini responsive, abbiamo bisogno di un sito web responsive con cui lavorare. In questo capitolo, costruiremo una pagina web di esempio aggiungendo due immagini che cambieranno a seconda del dispositivo dell'utente.

Fonte Immagine

Schermi retina

I display retina hanno il doppio dei pixel per pollice rispetto ai display a risoluzione standard.

Per rendere correttamente un'immagine su un dispositivo retina, l'immagine deve essere grande il doppio delle sue dimensioni di visualizzazione finali.

Per esempio, se vuoi aggiungere un'immagine di 500×250 pixel alla pagina, il file dell'immagine corrispondente deve essere di 1000×500 pixel.

Ecco un esempio.

                    
    <div class='section content'>
        <img class='illustration' src='asset/img/image-medium-alissa.svg' 
        style='max-width: 500px'/>
    <div'>

Immagini SVG responsive

Le immagini SVG (Scalable Vector Graphics) sono vettoriali, il che le rende naturalmente responsive e perfette per i display retina.

Aggiungiamo un'illustrazione SVG alla nostra pagina responsive.

                    
    <div class='section content'>
        <img class='illustration' src='images/illustration.svg' />
    <div'>

Per fare in modo che l'immagine si adatti correttamente ai diversi layout, aggiungiamo una regola CSS.

                    
    css :
    
    .illustration {                        
        width: 100%;
        max-width: 250px; /* Limitiamo l'immagine a 250px */
    }

Vediamo il risultato avremo chiaramente un immagine SVG di 250 x 250.

Immagini raster responsive (PNG, GIF, JPG)

Le immagini raster (PNG, GIF, JPG) sono definite pixel per pixel, rendendole più sensibili alla risoluzione dello schermo rispetto alle SVG.

Possiamo utilizzare l'attributo srcset per fornire diverse versioni di un'immagine a seconda della risoluzione dello schermo.

                    
    html: 

    <div class='illustration'>
        <img src='asset/img/image-small-alissa.png';
            srcset='asset/img/image-small-alissa.png 1x,;
            asset/img/image-medium-alissa.png 2x';
            style='max-width: 500px'/>
    <div'>

Vediamo il risultato

L'attributo srcset punta a una lista di file immagine alternativi, insieme alle proprietà che definiscono quando il browser dovrebbe usare ciascuno di essi.

L'attributo x1,x2 indicano i DPR quindi se x2 parliamo di un dispaly retina.

Ottimizzare immagini per la larghezza dello schermo

Per ottimizzare le immagini in base alla larghezza dello schermo, possiamo utilizzare srcset insieme all'attributo sizes.

                    
    html: 
    
    <div class='illustration'>
        <img src='asset/img/image-small-alissa.png'
            srcset='asset/img/image-medium-alissa.png 2000w,
            asset/img/image-medium-alissa.png 1000w';
            sizes='(min-width: 960px) 960px,
    <100vw'/'>

Vediamo il risultato

L'attributo sizes definisce una serie di media query insieme alla larghezza resa dell'immagine quando quella media query è in vigore.

Direzione artistica con <picture>

L'elemento <picture> e gli elementi <source> consentono di ottimizzare i layout inviando immagini completamente diverse all'utente a seconda del proprio dispositivo. 

                    
        html: 

        <div class='section header'>
            <div class='photo'>
                <picture>
                    <source media='(min-width: 401px)'
                               srcset='asset/img/image-medium-alissa.png' width='100%'/>
                    <source media='(min-width: 400px)'
                               srcset='asset/img/image-small-alissa.png' width='100%'/>
                    < src='asset/img/image-small-alissa.png'/>
                    </picture>    
            <div'>
        <div'>

Vediamo il risultato

Le immagini responsive possono sembrare piuttosto complicate, ma ci sono due problemi principali che stiamo cercando di risolvere:

  • Fare in modo che le immagini si adattino ai layout mobili rispettando le loro dimensioni intrinseche.
  • Evitare che l'utente scarichi file immagine inutilmente grandi.

Abbiamo utilizzato srcset per ottimizzare la risoluzione dello schermo, sizes; per definire le dimensioni dell'immagine in base al dispositivo, e l'elemento <picture> per il controllo manuale su quale file immagine viene visualizzato.

Questi approcci ti aiuteranno a presentare immagini ottimizzate per qualsiasi dispositivo, migliorando l'esperienza utente sul tuo sito web.
